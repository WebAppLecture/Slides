Polymorphie in JavaScript vor ES 6

  • Polymorphie bedeutet, dass man eigene Implementierungen bekannter Methoden erstellen kann
  • Können über das prototype Schlüsselwort die Basis einer Klasse definieren
  • Um dem Beispiel aus der Grafik zu folgen: 
    function Shape() { //Grundsätzliches festlegen, hier x und y Startpunkt
	this.X = 0; this.Y = 0; } //Wie immer public Eigenschaften mit this
Shape.prototype.Draw = function () { ... };//Draw von Shape als Standard festlegen
Rectangle.prototype = new Shape();//Vererbung erfolgt hier (Basis ist ein neuer Shape)
Rectangle.prototype.Draw = function () { ... };//Eigene Implementierung (zeichnet Rechteck)
Circle.prototype = new Shape();//Vererbung erfolgt hier (Basis ist ein neuer Shape)
Circle.prototype.Draw = function () { ... };//Eigene Implementierung (zeichnet Ellipse)
Triangle.prototype = new Shape();//Vererbung erfolgt hier (Basis ist ein neuer Shape)
Triangle.prototype.Draw = function () { ... };//Eigene Implementierung (zeichnet Dreieck)
